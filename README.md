# CBT Diary App

**CBT Diary App** — мобильное приложение для анализа эмоционального состояния на основе методов когнитивно-поведенческой терапии с элементами искусственного интеллекта.

Проект реализован как клиент-серверное приложение: мобильный клиент написан на **Flutter**, серверная часть — на **FastAPI**, база данных — **PostgreSQL**.

## О проекте

Приложение помогает пользователю структурировать тревожные или стрессовые ситуации по принципам когнитивно-поведенческой терапии. Пользователь проходит пошаговую КПТ-сессию в формате чата с ИИ-помощником, после чего итоговая запись сохраняется в дневник. На основе сохранённых записей формируется аналитика эмоционального состояния.

Также в приложении реализован каталог специалистов, роли пользователя, терапевта и администратора, переписка между пользователем и терапевтом, модерация анкет специалистов и система уведомлений.

## Backend architecture

Backend реализован на FastAPI и разделён на функциональные модули.

### Основные файлы backend

```text
main.py
```

Точка входа в FastAPI-приложение. Подключает роутеры, статические файлы и общие настройки backend.

```text
database.py
```

Настройка подключения к PostgreSQL через SQLAlchemy. Содержит `engine`, `SessionLocal`, `Base` и зависимость `get_db()`.

```text
models.py
```

SQLAlchemy-модели базы данных. Каждый класс соответствует таблице PostgreSQL.

```text
schemas.py
```

Pydantic-схемы для валидации входных данных и формирования ответов API.

```text
security.py
```

Логика безопасности: хеширование паролей, проверка пароля, создание JWT-токена, получение текущего пользователя.

```text
routers/
```

Папка с API-маршрутами. Каждый файл отвечает за отдельную функциональную область приложения.

```text
ai/
```

Модуль ИИ-помощника: промпты, guardrails, клиент LLM и логика генерации ответов для КПТ-сессии.


## Установка и запуск backend

Перейти в папку backend:

```bash
cd backend
```

Создать виртуальное окружение:

```bash
python -m venv venv
```

Активировать виртуальное окружение:

Windows:

```bash
.\venv\Scripts\activate
```

Установить зависимости:

```bash
pip install -r requirements.txt
```

Создать файл `.env` в папке backend.

Пример `.env`:

```env
DATABASE_URL=postgresql://postgres:password@localhost:5432/cbt_diary_db
SECRET_KEY=your_secret_key
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=1440

GOOGLE_CLIENT_IDS=your_google_client_id
LLM_API_KEY=your_llm_api_key
LLM_MODEL=your_llm_model
```

Запустить backend:

```bash
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

## Работа с авторизацией

В приложении используется JWT-авторизация.

1. Пользователь отправляет email и пароль.
2. Backend проверяет данные.
3. Backend возвращает `access_token`.
4. Flutter сохраняет токен в защищённом хранилище.
5. При следующих запросах `ApiClient` автоматически добавляет токен в заголовок:

```http
Authorization: Bearer <access_token>
```

Если backend возвращает ошибку `401 Unauthorized`, токен очищается.

## Роли пользователей

В приложении реализованы роли:

```text
user
therapist
admin
```

### user

Обычный пользователь приложения. Может проходить КПТ-сессии, вести дневник, смотреть аналитику, выбирать терапевтов и переписываться с ними.

### therapist

Специалист. Может заполнить профиль, загрузить сертификаты, отправить анкету на модерацию и переписываться с клиентами.

### admin

Администратор. Может модерировать анкеты терапевтов и просматривать сводную статистику.

## ИИ-помощник

ИИ-помощник используется для сопровождения пользователя по шагам КПТ-сессии.

Основная логика:

```text
SITUATION
↓
AUTOMATIC_THOUGHT
↓
EMOTIONS
↓
EVIDENCE_FOR
↓
EVIDENCE_AGAINST
↓
ALTERNATIVE_THOUGHT
↓
RESULT
↓
FINISHED
```

ИИ-помощник не ставит диагнозы, не заменяет психотерапевта и используется как инструмент для структурирования мыслей в формате КПТ-дневника.

## Аналитика

На основе дневниковых записей приложение формирует аналитику:

* количество сессий;
* количество завершённых сессий;
* количество записей дневника;
* динамика состояния по дням;
* частые когнитивные искажения;
* использованные техники;
* общий показатель устойчивости.

## Работа с файлами

Терапевт может загружать:

* фото профиля;
* сертификаты.

Файлы передаются на backend через `multipart/form-data` и сохраняются в папке uploads.
